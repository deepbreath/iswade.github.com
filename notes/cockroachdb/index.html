    cockroach

    对于工程项目，cockroachdb 是一个非常 优雅 的工程。 对于所有的架构，软件，工具，代码，各个方面都值得深入学习。 1. 自包含（分布式系统但只有一个发布的软件） 2. 自说明（伸缩性，强一致，生存能力） 3. 单实例（服务器、客户端同一个二进制文件） 4. 免配置（方便，易用） 5. 文档、博客清晰易读 6. RFC文档 7. 工程实践（推广） 8. 简单监控（易用性）

    使用 docker 运行

    1. 安装 docker（brew install docker）

    2. 启动 docker（在应用程序中启动）

    3. 检查 docker 是否运行正常

    docker version

    1. 获取镜像

    docker pull cockroachdb/cockroach:v1.1.5

    1. 启动（参数 -p 用于将内部端口映射为外部端口，然后就可以直接通过客户端访问数据库）

    docker run -p 8080:8080 -p 26257:26257 cockroachdb/cockroach-unstable start --insecure

    1. ./gopgcli -d db 就可以正常访问了

    编译安装

    方法一：下载源码

    ```go get -d github.com/cockroachdb/cockroach go get -d github.com/cockroachdb/cockroach

GOFLAGS      := -i -a -v -gcflags="all=-N -l" -tags nopkcs11 -ldflags='-linkmode internal'

### 方法二：下载打包源码

依赖工具：c++编译器、Go、Bash、CMake、Autoconf

    wget https://binaries.cockroachdb.com/cockroach-v1.1.2.src.tgz | tar xvz
    cd cockroach-v1.1.2

    make build
    make install

## 两种调试方式
首先要编译：
加快 cockroachdb 的调试速度，将下面的文件

调试技巧2：快捷键单步调试当前用鼠标点击的方式可以，快捷键会导致卡死

## 运行单节点

    cockroach start --insecure --background

如果是初次运行，会在当前文件夹下面创建一个目录：`cockroach-data`
后续运行上面的命令时，默认使用上面创建的目录。

## 启动集群

启动三个节点的命令如下所示：

    cockroach start --insecure --background\
        --store=cockroach-data \
        --host=localhost \
        --port=26257 \
        --http-port=8080

    cockroach start --insecure --background\
        --store=cockroach-data2 \
        --host=localhost \
        --port=26258 \
        --http-port=8081 \
        --join=localhost:26257

    cockroach start --insecure --background\
        --store=cockroach-data3 \
        --host=localhost \
        --port=26259 \
        --http-port=8082 \
        --join=localhost:26257

第一个命令跟单节点一致，后续多了一个 join 参数，用于跟第一个节点建立关系。

可以用于验证副本自动均衡等集群相关特性。
