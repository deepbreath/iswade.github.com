    哈希表

    哈希表实现了关联数组抽象数据类型把键映射到值。

    使用了哈希函数来计算数组的偏移，在这里可以找到期望的值。

    静态哈希表

    分配一个巨大的数组，每个元素都有一个槽来记录。

    为了找到入口，通过对 key 求元素个数的模，这样就可以找到数组的偏移。

    假设

    提前知道元素的数量。

    每个 key 是唯一的。

    完美的哈希函数。如果 key1 \ne key2，则 hash(key1) \ne hash(key2)。

    哈希表

    设计策略 #1：哈希函数

    • 如何将很大范围的 key 空间映射到小的域。
    • 速度与碰撞比率的权衡。

    设计策略 #2：哈希模式

    • 如何在哈希后处理 key 碰撞
    • 分配大哈希表与额外的查找/插入 key 操作之间的权衡

    哈希函数

    我们不想使用加密哈希函数用到 join 算法中。

    我们想使用速度很快并且低碰撞率的函数。

    1. MurmurHash (2008)

      Designed to a fast, general purpose hash function.

    2. Google CityHash (2011)

    Based on ideas from MurmurHash2 → Designed to be faster for short keys (<64 bytes).

    1. Google FarmHash (2014)

    Newer version of CityHash with better collision rates.

    1. CLHash (2016)

    Fast hashing function based on carry-less multiplication.
